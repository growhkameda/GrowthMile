# スライドデザインテーマ

本プロジェクトのスライド資料は、以下の2テーマのいずれかで作成する。
**作成に入る前に必ずユーザーへどちらのテーマにするか確認すること**（省略しない）。
過去の実績: 技術者向け共有資料 = tech-dark ／ 手順書・オンボーディング資料 = enterprise-light。

## 共通ルール（両テーマ）

- レイアウト: 16:9（10 x 5.625 インチ）。余白は上下左右 0.5 インチ以上
- フォント: 日本語 `Yu Gothic` ／ コード・数値・ID `Courier New`
- 箇条書きは日本語資料の慣例に合わせ行頭「・」を使う（bullet:true の点は小さく浮いて見えるため）
- コマンドは専用ボックスに `$ コマンド   # コメント` 形式で記載
- 1行が長い日本語は「1文字だけの孤立改行」が起きやすい。文言を短縮して1行に収める
- アクセントバー・見出し下線・端ストライプは使用禁止（AI感が出るため）
- 図版: アイコンは react-icons を PNG 化して円形バッジに載せる

## テーマ 1: tech-dark（ダーク系テック）

開発者向けの技術共有・アーキテクチャ説明・社内勉強会向け。

| トークン | 値 | 用途 |
| --- | --- | --- |
| BG | `0F172A` | 全スライド背景（ダーク統一） |
| CARD | `1E293B` / CARD2 `24334D` | カード・パネル |
| CODE | `0B1120` | コマンドボックス背景 |
| TXT | `F1F5F9` / MUT `94A3B8` | 本文 / 補足 |
| ACCENT | `2DD4BF`（teal） | 見出しキッカー・強調・アイコン円 |
| ACCENT2 | `FBBF24`（amber） | 人間ゲート・警告のみ（多用禁止） |

- 見出し: 上に MONO 大文字キッカー（teal・charSpacing 3）、下に白の太字タイトル
- カードは fill CARD + 影（blur 8 / offset 3 / opacity 0.35）
- コマンド文字は teal 太字、`$` と `#` コメントは `64748B`
- タイトルスライドの signature: mono のディレクトリツリーやフローリストをカードで右側に

## テーマ 2: enterprise-light（エンタープライズ・かっちり）

手順書・オンボーディング・顧客提出・報告書向け。SIer 定番フォーマット。

| トークン | 値 | 用途 |
| --- | --- | --- |
| PAPER | `FFFFFF` | 内容スライド背景 |
| NAVY | `1F3864` | 見出し・表ヘッダー・強調 |
| NAVYBG | `1B3155` / NAVYP `2A4A7F` | 表紙・最終頁の背景 / その上のパネル |
| INK | `333F52` / MUT `6B7280` | 本文 / 補足 |
| LINE | `D5DBE4` / TINT `EEF2F8` | 罫線 / ゼブラ行・チップ背景 |
| CODEBG | `F4F6F8` | コマンドボックス背景 |
| WARN | bg `FFF4E5` fg `B45309` | 注意カード |
| DANGER | bg `FCEBEA` fg `C0392B` | 禁止事項カード |

- サンドイッチ構成: 表紙・最終頁のみ紺背景（白文字）、内容ページは白背景
- 表紙に文書管理情報（対象者・所要時間・作成日・版数・管理番号 GM-DOC-xxx）を必ず入れる
- 見出し: 紺の正方形に白のSTEP番号 + キッカー + 紺太字タイトル
- 表は「紺ヘッダー行（白文字）+ ゼブラ行（TINT/白交互）+ 細罫線」で作る
- カードは白 fill + LINE 罫線 + ごく薄い影（opacity 0.10）
- フッターに資料名 + 文書番号 + ページ番号（グレー小字）

## 移譲（ポータビリティ）

- 本スキルはフォルダ単体で自己完結しており、`slide-design-themes/` ごとコピーすれば他プロジェクトで使える
- `dev-diagrams` スキルは同じトークンの複製（themes.md）を同梱している。両方を移譲した場合は本スキルを正として同期すること

## 実装メモ

- pptxgenjs で生成する（Cowork / Claude Code の pptx スキル手順に従う）
- 既存の実装例: docs/agentic/ の3資料（claude-config-rationale = tech-dark ／ onboarding-setup-guide = enterprise-light）
- 生成後は必ずビジュアルQA（画像化 → 検査）を行い、テキストあふれ・孤立改行を修正する
