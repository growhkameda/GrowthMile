# フロントエンド・UIデザイン原則

> [INFO] 汎用コアルール。具体的なデザイントークン値・フォント名はプロジェクト側（`project-stack.md`・デザインシステム文書）で定義する。

## デザイントークン

- 色をハードコードしない。必ず CSS 変数（デザイントークン）を使用する
- テーマ切替（light / dark）は `[data-theme]` 属性等による変数定義の切り替えで実現する
- 新しい色が必要な場合は、デザインシステム定義に追記してから使用する

## タイポグラフィ・形状

- フォント・タイプスケール・角丸・余白はプロジェクトのデザインシステム定義に従う
- 数値・ID・時刻・コードは等幅フォントを使用する

## 禁止事項

- [NG] `box-shadow` / glow エフェクトの乱用
- [NG] 派手なアニメーション（原則 transition 0.3s 以下）
- [NG] ネオンカラー（純粋な `#00FFFF` 等）の使用
- [NG] 汎用色名の直接指定（`color: blue` / `background: gray` 等）

## コンポーネント設計

- 単一責務: 1コンポーネント1責務。肥大化したら分割する
- 状態は必要最小のスコープに置く。グローバル状態ストアには純粋な UI 状態のみを入れ、サーバー状態を混ぜない
- プレゼンテーション（見た目）とロジック（データ取得・変換）を分離する

## アクセシビリティ・品質フロア

- キーボードフォーカスを可視化する
- 画像に alt、フォーム要素に label を付ける
- モバイル幅までレイアウトが崩れないレスポンシブ対応を品質フロアとする
- `prefers-reduced-motion` を尊重する
