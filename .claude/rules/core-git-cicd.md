# Git・CI/CD 規約

> [INFO] 汎用コアルール。CI 実装（ジョブ構成・使用ツール）の固有ルールは `project-ops.md` を参照。

## コミット・PR

- コミットメッセージは Conventional Commits（`feat:` `fix:` `docs:` `refactor:` `test:` `chore:`）に従い、英語で書く
- 1コミット1論理変更。無関係な変更を混ぜない
- [NG] `git push --force`（共有ブランチの履歴書き換え禁止）
- PR はテンプレートに従い、CI の品質チェックをすべてパスさせること

## CI 設定ファイル（YAML）の書き方

- `run:` 内のクォートされていない行の途中に「半角コロン + 半角スペース」を含めない（辞書と誤パースされ `mapping values are not allowed` になる）。日本語メッセージは全角コロンで回避する
  - [NG] `- echo "エラー: プレースホルダーが存在します"`
  - [OK] `- echo "エラー：プレースホルダーが存在します"`
- `|` / `>` の複数行ブロックは、親要素から正確に一段深いインデントで統一する
- `*.yml` `*.yaml` `*.sh` の改行コードは、どの OS 環境でも必ず **LF** で保存する
- `${{ }}` 式とシェル変数 `$` が同一行に混在する場合、意図しない展開が起きないか必ず確認する

## CI ジョブ設計

- アーティファクト出力ディレクトリは `run:` の先頭で `mkdir -p` する
- テストが途中クラッシュした場合に備え、レポートのスタブを `if: always()` ステップで生成する
- E2E 用ランナーイメージとテストフレームワークのバージョンは厳密一致させる（caret `^` によるレンジ指定禁止）
- `paths` フィルタのみに依存せず、主要ブランチへの `push` でも E2E が走るようトリガーを明示する

```yaml
# [OK] PR と main/develop の push で E2E を実行する例
on:
  pull_request:
  push:
    branches: [main, develop]
```
