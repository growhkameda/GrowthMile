# モデル選定ガイドライン

このプロジェクトでは **Claude Sonnet 4.6 をデフォルト** として使用し、以下の条件に該当する場合のみ **Claude Opus 4.6 へのエスカレーションをユーザーに提案** すること。

## Sonnet 4.6 で対応するタスク（デフォルト）

- 要件定義・仕様書・ユーザーストーリー・ADR の作成
- 標準的なコーディング・補完・ドキュメント生成
- フロントエンド設計・UI 実装・Tailwind CSS スタイリング
- UIデザインモックアップ作成（HTML/CSS）
- API仕様確認・DB 設計レビュー（軽微な変更）
- 一般的なパイプライン・標準 QA・テストケース生成

## Opus 4.6 が必要なタスク（エスカレーション対象）

以下のタスクが含まれると判断した場合、**作業開始前にユーザーへモデル切り替えを提案** すること。

- 大規模リファクタリング（複数モジュール・コードベース全体の改修）
- セキュリティ審査・脆弱性分析・クリティカルパスの設計（`code-reviewer` SubAgent は常に Opus）
- 複雑なデバッグ・インシデント対応（根本原因が不明な障害）
- マルチエージェント設計・長時間自律タスク管理
- PhD レベルの技術的難問・学術的推論が必要な場面

## ユーザーへの提案フォーマット

Opus 4.6 が適切と判断した場合、以下のフォーマットで提案すること（1回のみ、強制しない）：

> このタスクは [理由] を含むため、**Claude Opus 4.6 での実行を推奨します**。
> Claude Code のモデルを `claude-opus-4-6` に切り替えて再度指示してください。
> Sonnet 4.6 のままでも対応可能ですが、精度とコストのトレードオフを考慮してください。

## モデル設定

- Claude Code のデフォルトモデルは `.claude/settings.json` の `model` フィールドで管理する。
- コスト目安: Opus 4.6 は Sonnet 4.6 の **5倍** のコスト（入力 $15 vs $3 per 1M tokens）。
- ハイブリッド戦略: 80-90% のタスクは Sonnet で処理し、コスト 60-80% 削減しながら品質を維持する。
