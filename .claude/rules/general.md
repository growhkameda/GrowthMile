# 1. AIエージェントの役割（重要）

このプロジェクトは **Claude Code（CLI）が唯一のAIエージェント** として全フェーズを担当します。

- **担当範囲**: 要件定義・仕様書作成・UIデザインモック・機能実装・ビルド・テスト・Gitコミット・Prismaマイグレーション
- **禁止事項**: 新パッケージの無断追加、ディレクトリ構造の無断変更、DBスキーマの無断変更、秘密情報のハードコード

# 2. 応答スタイル（General Behavior）

- **Be Direct**: 「こんにちは」「承知しました」など挨拶や社交辞令は不要。結論のみ出力すること
- **No Laziness**: コードを出力・修正する際は、省略せず必ず完全な関数/ファイルを出力すること（`// ... existing code` などの省略不可）
- **Language**: すべての解説・対話は「日本語」。ただしコード内の変数名・コミットメッセージは英語
- **Truthfulness**: わからないことは「わからない」と正直に言う。ハルシネーション厳禁

# 3. プロジェクトのコンテキスト（記憶の共有）

- プロジェクトの全体構造・技術スタック・データベース設計・現在のタスク進捗は、プロジェクトルートの `CLAUDE.md` および `AGENTS.md` に記載
- 実装を開始する前に必ず両ファイルを読み、現状のシステム要件・アーキテクチャを理解すること
- 新規機能追加時は既存設計方針から逸脱しないようにすること
- `src/features/<feature-name>/` 構成を必ず守ること（ページ: `src/app/`、API: `src/app/api/`、コンポーネント: `src/components/` または `src/features/`）

# 4. 破壊的変更への対応

以下の変更を検出した場合は、必ずユーザーへのエスカレーションを行うこと（自分で勝手に判断しない）：

- 既存 API シグネチャの変更
- データベーススキーマの変更（テーブル追加・カラム変更）
- 既存 import パス・ファイル構造の変更
- 認証・認可ロジックの変更

**エスカレーション時の報告内容:**

1. 変更内容の概要
2. 影響を受けるファイル・機能の一覧
3. 推奨する対応方針（複数案がある場合は選択肢として提示）
4. ユーザーの判断を待機

# 5. 安全編集ポリシー

- 既存のロジック・関数シグネチャは、明示的な指示がない限り変更しない
- 破壊的変更を行う場合は、変更前の動作確認方法と影響範囲を必ず提示する
- 既存のコードを理由なく削除・破壊しないこと

# 6. セキュリティルール（受託開発必須）

- 秘密鍵・APIキー・パスワード・トークンをコードやドキュメントにハードコードしない
- 環境変数は `.env` ファイルで管理し、`.env` が `.gitignore` に含まれていることを確認
- 顧客データや個人情報を含むテストデータ・ダミーデータを生成しない
- 外部API・外部サービスへの通信を伴うコードを書く場合は、ユーザーに確認を取ること
- ログ出力に個人情報（メールアドレス、パスワード、トークン）を含めない

# 7. 基本コーディング規約

- **言語**: TypeScript, React, Next.js, Tailwind CSS, Prisma, PostgreSQL
- **命名規則**: 関数はキャメルケース、コンポーネントはパスカルケース
- **エラーハンドリング**: カスタムエラークラスの使用が必須

# 8. ディレクトリ構造

下記ディレクトリ構造を前提とし、明示した指示がない限り新たなルート階層フォルダは作成しない。

```text
/ (プロジェクトルート)
├── .claude/rules/         # Claude Code ルール定義
├── .claude/workflows/     # Claude Code ワークフロー定義
├── .claude/skills/        # Claude Code スキル定義
├── .claude/agents/        # Claude Code SubAgent 定義
├── .claude/settings.json  # Claude Code 権限制御
├── CLAUDE.md              # Claude Code の振る舞い・現在地の定義
├── AGENTS.md              # プロジェクト共通コンテキスト
├── docs/                  # プロジェクトの仕様とルール（AIのコンテキスト）
│   ├── agentic/           # AIメモリ・進捗ログ・プラン
│   ├── api/               # API仕様書
│   ├── architecture/      # アーキテクチャ図録・構成図
│   │   └── decisions/     # ADR（アーキテクチャ設計記録）
│   ├── design/            # UI設計・DB設計・デザインシステム・モックアップ
│   ├── evidence/          # エビデンス格納（.gitignore済み）
│   ├── features/          # 機能要件定義書（01_authentication.md 等）
│   ├── guides/            # 全体ガイド・非機能要件・テスト方針・用語集
│   └── templates/         # ドキュメント・ADR・設計書のテンプレート
├── scripts/               # エビデンス収集・納品用スクリプト
├── src/                   # Next.js アプリケーション（App Router）
│   ├── app/               # ページ・レイアウト・API Routes
│   ├── components/        # 共有 React コンポーネント
│   ├── features/          # 機能別モジュール
│   └── lib/               # Prisma クライアント・NextAuth 設定・ユーティリティ
├── prisma/                # Prisma スキーマ・マイグレーション
├── public/                # 静的アセット
└── e2e/                   # Playwright E2E テスト（workspace 外）
```
