# ガードレール（安全・エスカレーション）

> [INFO] 汎用コアルール。AI駆動開発における「AIが自律で越えてはいけない線」を定義する。

## 破壊的変更のエスカレーション（必須）

以下を検出した場合、自分で判断せず必ずユーザーへエスカレーションする:

- 既存 API シグネチャの変更
- データベーススキーマの変更（テーブル追加・カラム変更を含む）
- 既存 import パス・ファイル構造の変更
- 認証・認可ロジックの変更
- `rm -rf` 等の破壊的コマンドの提案

**エスカレーション時の報告内容:**

1. 変更内容の概要
2. 影響を受けるファイル・機能の一覧
3. 推奨する対応方針（複数案がある場合は選択肢として提示）
4. ユーザーの判断を待機

## 安全編集ポリシー

- 既存のロジック・関数シグネチャは、明示的な指示がない限り変更しない
- 既存コードを理由なく削除・破壊しない
- 破壊的変更を行う場合は、変更前の動作確認方法と影響範囲を必ず提示する

## 禁止事項

- [NG] 新しいパッケージ・依存関係の無断追加（提案のみ可）
- [NG] ディレクトリ構造の無断変更・新規ルート階層フォルダの無断作成
- [NG] 指定外のフレームワーク・ライブラリの導入
- [NG] 秘密鍵・APIキー・パスワード・トークンのハードコード（環境変数 `.env` を使用）
- [NG] `.env` / `.env.*` の内容の自動探索・解析（必要な値は人間が明示的にチャットへ貼る）
- [NG] テストデータ・ダミーデータへの実在個人情報の使用
- [NG] ログ出力への個人情報（メールアドレス・パスワード・トークン）の含有
- [WARN] 外部 API・外部サービスへの通信を伴うコードは、実装前にユーザーへ確認を取る
