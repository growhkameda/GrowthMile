# セキュリティ原則

> [INFO] 汎用コアルール。受託開発を含むすべてのプロジェクトで必須。

## 秘密情報の管理

- 秘密鍵・APIキー・パスワード・トークンをコード・ドキュメント・ログにハードコードしない
- 環境変数は `.env` で管理し、`.env` が `.gitignore` に含まれていることを確認する
- 設計書・レビュー記録・エビデンスに秘密情報の実値・実在個人情報を記載しない

## 入力起点の脆弱性対策

- **SQL インジェクション**: 必ずパラメータバインド（ORM / プリペアドステートメント）を使用する
- **XSS**: フレームワークの安全な描画機構を使用する。エスケープを迂回する API（`dangerouslySetInnerHTML` 等）は正当な理由とサニタイズが必須
- **パストラバーサル**: ユーザー入力をファイルパスに直接使わない
- **SSRF**: ユーザー入力の URL への外部リクエストは許可リスト方式にする

## 認証・認可

- 認可チェックは必ずサーバー側で行う（UI の出し分けはセキュリティ境界ではない）
- 認証が必要なルートは一元的なミドルウェア等で保護し、ページ個別チェックの漏れを防ぐ
- セッション・トークンの有効期限と失効手段を設計に含める
- IDOR 対策: リソース取得時は「認証済みか」だけでなく「そのユーザーの所有物か」を検証する

## その他

- CORS は必要最小限のオリジンのみ許可する
- 依存パッケージの追加時は保守状況・既知脆弱性を確認する（無断追加は禁止）
- セキュリティに関わる設計判断はセキュリティガイド文書（`docs/guides/security.md` 等）に明記する
